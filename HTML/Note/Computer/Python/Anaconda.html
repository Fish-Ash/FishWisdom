Anaconda

用于管理虚拟环境。

因为Python代码会用到很多库嘛，如果不用虚拟环境的话，就会出现一个环境下有无数个库的情况（哪怕用不到），还有一种情况就是Python的版本和库的版本对应不上，这就是为什么要建虚拟环境。

以下指令均为终端指令

查看现有的虚拟环境列表 

            conda info -e
创建虚拟环境 

            conda create -n EnvName python=x.x

其中EnvName为自己取的虚拟环境的名字，x.x为你设定的Python的版本。

激活虚拟环境 

            conda activate EnvName

成功激活后路径前会有一个括号显示虚拟环境的名称。

退出当前虚拟环境 

            conda deactivate
删除虚拟环境 

            conda remove -n EnvName --all
查看当前环境下有哪些库 
            
            conda list
在当前环境下安装库 

            conda install RequirementName
